# Módulo 1 — HTML / JS / CSS (enunciado)

Objetivo: entregar una versión corregida y mejorada de la pequeña aplicación "Contact List" ubicada en este directorio.

## Enunciado de la práctica:
- Analiza el código existente, identifica fallos funcionales, malas prácticas y posibles problemas de seguridad.
- Aplica las correcciones necesarias para que la aplicación funcione de forma fiable y segura.
- Añade pruebas básicas y documentación que expliquen los cambios realizados y las decisiones tomadas.

## Requisitos operativos:
- La aplicación debe permitir añadir, listar y eliminar contactos correctamente desde el navegador.
- Debes incluir una breve justificación de los cambios en un archivo `CHANGES.md`.

## Comandos y ejecución:
- Abrir `module-1/index.html` en el navegador para inspeccionar el funcionamiento.
- Puedes añadir scripts, herramientas o tests locales si lo consideras necesario.

## Criterios de evaluación:
- Funcionamiento correcto según el enunciado (añadir, listar, eliminar).
- Mejora de la calidad del código: organización, legibilidad y modularidad.
- Mitigación de riesgos y bugs detectados (seguridad, validación de entradas).
- Pruebas y documentación que demuestren y expliquen las correcciones.

Tiempo recomendado: 30–50 minutos.
